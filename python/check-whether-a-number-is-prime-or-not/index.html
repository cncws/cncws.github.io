优化 is_prime 函数

示例中用到 is_prime 函数来检测一个数是否为素数。趁此机会学习了一下 is_prime 的优化方案。原文

上述 is_prime 并不通用，因为它没有考虑边界情况（2 是素数）。考虑到函数的输入 n 远远大于 2，示例中这样做没什么问题。

下面单独讨论如何更高效判断一个数是否是素数。根据素数的定义，素数只能被 1 和自身整除，所以最直白的算法是依次检测能否被 2 3 ... n-1 整除。

优化的做法是排除检测某些不可能整除的值，例如前文见过的跳过所有偶数以及大于 \(\lfloor \sqrt{n}\rfloor\) 的整数。

更进一步，我们可以利用下述结论：所有的素数，都是 \(6k\pm 1\) 的形式，除了2和3。因此，如果一个数不能被 2 或 3 整除的话，那么在接下来的检测中，我们只需检测能否被 \(6k\pm 1\) 整除即可。因为所有的整数都能表示为 \(6k\pm i,\quad i=-1,0,1,2,3,4\)，其中 \(6k+0\)、\(6k+2\) 和 \(6k+4\) 能被2整除，\(6k+3\) 能被 3 整数。

多线程与多进程

对于计算是否是素数这种 CPU 密集型作业，多线程没有优势，甚至比按序计算还慢一点点。而多进程则明显要快很多。
