堆和堆排序

更新于 2020-07-15 分类于 笔记 Valine：

堆

堆（Heap）始于J. W. J. Williams在 1964 年发表的堆排序（heap sort）。

堆是一种基于树的数据结构，一棵满足以下性质的完全树可称为堆。

给定堆中任意节点 P 和 C，若 P 是 C 的母节点，那么 P 的值小于等于（或大于等于）C 的值。

  1. 若 P 的值小于等于 C 的值，称为最小堆（min heap）。
  2. 若 P 的值大于等于 C 的值，称为最大堆（max heap）。

在堆中最顶端的节点称为根节点（root node），根节点本身没有母节点（parent node）。

常见的堆有二叉堆、斐波那契堆等。

操作

一个堆应该支持以下常见操作（参考了维基百科）。

  • peek()：返回堆顶元素。
  • pop()：返回并移除堆顶元素，需要调整堆。
  • append(x)：添加一个新的元素​，需要调整堆以满足堆的性质。
  • replace()：相当于一次 pop 之后 append。但比分别执行更效率，因为只需要调整一次堆。
  • heapify()：创建一个堆。只需要调用一次。
  • merge(other)：合并 other 中的元素到堆。
  • size()：返回堆中元素个数。
  • is_empty()：返回是否是空堆。
  • increase_key(i, x)：增大元素的值。
  • decrease_key(i, x)：减小元素的值。
  • delete(i)：删除指定的一个元素。
  • sift_up(i)：不断上移元素，直到满足最小堆的性质。
  • sift_down(i)：不断下移元素，直到满足最大堆的性质。

实现

堆通常用数组（静态或动态）实现，并且不需要保存指针。

树的根节点是 A[0]，给定一个节点的下标 i，通过计算可以得到它的母节点、左孩子和右孩子节点。

维护堆的性质

以最大堆为例，如何维护一个数组使之满足最大堆性质。考虑下面这种情况。

  • 前提条件：根节点为 left_child(i) 和 right_child(i) 的二叉树都是最大堆。

A[i] 有可能小于其孩子节点，这违背了最大堆的性质。sift_down() 方法通过让 A[i] 在最大堆中“逐级下降”，从而使下标为 i 的子树重新遵循最大堆的性质。

关于逐级下降正确性的思考：满足前提条件的情况下，

假设根节点 i 有子节点 x, y，其中值最大的是子节点 x。交换 i 与 x 的值，现在 A[i] > A[x]，满足最大堆的性质。但 x 的值相比交换前变小了，所以可能小于其子节点的值。

现在节点 x 可能小于其子节点的值，其子节点均大于或等于下一层子节点的值。符合前提条件，所以对 x 递归调用 sift_down。

sift_down 的时间复杂度：

在一棵大小为 n ​的子树上，运行 sift_down 的时间 T(n) 包括：比较三个节点大小关系以及交换值的时间代价 \(\Theta(1)\)，加上在一棵子树上运行 sift_down 的时间。子树的大小至多为 2n/3（最底层正好半满的时候，即最后一层左子树节点填满，右子树节点是空的），由此可以得到刻画程序运行时间的递归式：\(T(n)\le\Theta(1)+T(2n/3)\)。解得 \(T(n)=O(\lg{n})\)。

也就是说，对于一棵树高为 h 的节点来说，sift_down 的时间复杂度是 \(O(h)\)。

关于一棵子树的大小至多为 2n/3 的由来。

首先明确关于树的两个性质：

  1. 如果以某个节点为根节点的树是完全树，那么它的左子树大小大于等于右子树大小。因为完全树的最后一层是从左到右填入的。
  2. 如果一棵满二叉树的最后一层节点数目为 n，那么这个树的大小为 2n-1。因为满二叉树第 i(i>=0) 层节点数量是 \(2^i\)。

根据性质 1，只需要考虑左子树最多为母树的几分之几即可。

假设一个完全树的大小为 n，它的左子树大小为 i，比值 i/n 根据最底层节点数目而变动。观察最底层节点数目从 0 逐渐增长时该比值的变化情况。

  • 当新增节点属于左子树时，比值由 i/n 变成 (i+1)/(n+1)，比值增大。

  • 当新增节点属于右子树时，比值由 i/n 变成 i/(n+1)，比值减小。

因此当左子树恰好填满（最底层恰好半满）时，比值最大。

设此时最低层节点个数为 x，注意这 x 个节点正好填满左子树最底层，则根据性质 2，i=2x-1，\(n=2*i+1-x=3x-1\)。

所以得到 \(i/n=(2x-1)/(3x-1)\le2/3\)。即大小为 n 的树中，子树的大小至多为 2n/3。

建堆

同样以最大堆为例，可以用自底向上的方法调用过程 sift_down，将一个数组转换为最大堆。过程 heapify 对树中非叶子节点都调用一次 sift_down。

可以使用如下的循环不变量验证该过程的正确性。

每一次 for 循环开始前，节点 i, i+1, ..., size() 都是一个最大堆的根节点。

需要证明此循环不变量在循环开始前，迭代时以及结束后都成立。

  • 初始化：第一次迭代前，i 之后的节点都是叶节点，因此它们都是最大堆的根节点。
  • 保持：注意到节点 i+1​ 始终比i先调用 sift_down 过程，根据循环不变量，它们都是最大堆的根节点。
  • 终止：终止时 i=-1​，根据循环不变量节点 0,1,...,size() 都是最大堆的根节点，特别的，0 是堆的顶点。

之前分析了 sift_down(A, i) 的运行时间与节点 i 所处高度有关，在建堆过程中，不同的节点所处高度可能不同，想准确计算建堆的时间复杂度利用了下面所示的树的特性。

  • 一棵含有 n 个元素的完全树，在高度 h 处，最多有 \(\lceil{n/2^{h+1}}\rceil\) 个元素。根节点所处高度为 0。

根据这条性质，运行时间可以刻画为

\[ \begin{aligned}T(n) &=\sum_{h=0}^{\lfloor\log_{2}n\rfloor}\lceil{n/2^{h+1}}\rceil*O(h)\\ &=O(n\sum_{h=0}^{\lfloor\log_{2}n\rfloor}\frac{h}{2^h}) \end{aligned}\\ \begin{aligned} \because&\sum_{k=0}^{\infty}kx^k=\frac{x}{(1-x)^2}(|x|\le1)\\ \therefore&\sum_{h=0}^{\infty}\frac{h}{2^h}=\frac{1/2}{(1-\frac{1}{2})^2}=2 \end{aligned} \]

将其带入可得

\[ \begin{aligned}T(n) &=O(n\sum_{h=0}^{\infty}\frac{h}{2^h})\\ &=O(n) \end{aligned} \] 由此可得出结论，heapify 可以在线性时间内将一个数组转换成一个堆。

其他方法

  • sift_up：与 sift_down 相反，当节点 i 的值大于其母节点的值，而其他节点均满足堆的性质时，通过此过程将节点i不断上移，直到满足堆的性质。
  • increase_key：当堆中某个节点的值变大时，可能大于其母节点，与上述情况相同。
  • decrease_key：与sift_down的情况相同。
  • replace：更新后的值大于原来的值，调用 increase_key；小于原来的值，调用 decrease_key。
  • append：先添加一个值为无穷小的元素到堆的末尾，然后调用 increase_key 修改值。
  • peek：第一个元素就是堆顶元素，直接返回即可。
  • pop：将堆顶元素保存，然后用最后一个元素替换堆顶元素，删除最后一个元素。此时堆顶元素可能违背堆的性质，对其调用 sift_down。或者调用 decrease_key 将堆顶元素的值减小到最后一个元素值，两者调用的过程时一样的。
  • delete：与 pop 过程相同，pop 就是特殊的 delete 过程。
  • merge：逐次调用 append 添加另一个堆中的元素。
  • meld：取出两个堆中的元素，合并成新的列表，调用 heapify 创建堆。

堆排序

堆排序（Heapsort）是一种比较排序算法。堆排序在 1964 年首次被J. W. J. Williams提出，同时也是堆这种数据结构首次出现。堆排序是一种原址（In-place）、非稳定（NOT stable）排序算法。

堆排序可以看作是选择排序的优化版本。堆排序与选择排序类似，将数组分成两个部分，未排序部分和已排序部分。算法每次从未排序部分提取最大值放入已排序部分，当未排序部分为空时，排序结束。

选择排序通过一次数组遍历（时间复杂度 \(\Theta(n)\) 获取最大值。而堆排序通过堆这种数据结构快速获取最大值。获取的时间复杂度是 \(\Theta(1)\)，但获取之后要花费 \(\Theta(\lg{n})\) 时间调整堆，因此总运行时间为 \(\Theta(1)+\Theta(\lg{n})=\Theta(\lg{n})\)。由此可以看出堆排序相比于选择排序改进的点。

堆排序的运行时间分为两部分，建堆的时间 \(O(n)\) 和 n-1 次调整堆的时间 \(O(\lg{n})\)。所以总的运行时间是

\(T(n)=O(n)+(n-1)O(\lg{n})=O(n\lg{n})\)。
