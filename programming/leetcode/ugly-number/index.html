Note:

  1. 1 is typically treated as an ugly number.
  2. n does not exceed 1690.

Dynamic Programming

The first ugly number is 1. New ugly number can be computed by known ugly numbers.

For example, once we get a new ugly number x, then we add x*2, x*3, x*5 to candidate set. And then pick the minimum number as the next ugly number, replace the procedure.

If n is large, the candidates can be very large. We can see the candidates are x * 2, y * 3, z * 5 where x, y, z are all ugly numbers. x = y = z = 1 at the beginning. Once a candidate (say 2) is chosen, we replace the corresponding ugly number (x=1) by the next ugly number (x=2, 2 is the succeed ugly number of 1).
