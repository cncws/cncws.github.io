  • 1 <= m, n <= 100.
  • It's guaranteed that the answer will be less than or equal to 2 * 10 ^ 9.

Dynamic Programming

dp[i][j] denotes the number of possible unique paths that from (i, j) to (m, n). Indices start from 1 here.

  • i = m, robot can only move right. So, dp[m][j] = 1.
  • j = n, robot can only move down. So, dp[i][n] = 1.
  • i < m and j < n, robot can move down or right. When move right, robot has dp[i][j + 1] unique paths. When move down, robot has dp[i + 1][j] unique paths. So, dp[i][j] = dp[i][j + 1] + dp[i + 1][j].

Also, you can denote dp[i][j] as the number of possible unique paths that from (1, 1) to (i, j). Then,

  • dp[1][j] = dp[i][1] = 1.
  • dp[i][j] = dp[i - 1][j] + dp[i][j - 1].
  • dp[m][n] is the answer of problem.

Dynamic Programming

Base cases:

  • If grid[i][j] == 1, which means a obstacle, dp[i][j] = 0.
  • i = m, robot can move right only if grid[i][j + 1] == 0. So, dp[i][j] = dp[i][j + 1].
  • j = n, robot can move down only if grid[i + 1][j] == 0. So, dp[i][j] = dp[i + 1][j].

Induction rules:

  • If grid[i][j] == 1, dp[i][j] = 0.
  • If grid[i][j] == 0, dp[i][j] = dp[i + 1][j] + dp[i][j + 1].
