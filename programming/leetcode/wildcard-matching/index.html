Wildcard Matching

更新于 2020-05-23 分类于 编程 Valine：

Given an input string (s) and a pattern (p), implement wildcard pattern matching with support for '?' and '*'.

The matching should cover the entire input string (not partial).

Note:

  • s could be empty and contains only lowercase latters a-z.
  • p could be empty and contains only lowercase latters a-z, and characters like ? or *.

Dynamic Programming

Similar to the problem Regular Expression Matching.

dp[i][j] denotes whether s[i..m) matches p[j..n). Index of strings start from 0.

Assuming m is the length of string s, n is the length of pattern p.

Base cases:

  • origin: dp[m][n] = true. s[m:] == p[n:] == '', they do match.
  • last row: dp[m][j], expect for string p is *, which can match empty sequence, otherwise all false.
  • last column: dp[i][n] = false. s[i:] != '' but p[n:] == '', they can't match.

There also exists another variation, where dp[i][j] denotes whether s[0..i-1] matches p[0..j-1]. It's similar to the solution above, but be careful about the indices.
