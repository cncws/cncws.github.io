幸运数

更新于 2020-07-29 分类于 编程

求区间 [l, r] 内的幸运数的个数。幸运数定义为，将相邻位上数字差的绝对值拼成下一个数，重复该操作直到只剩 1 位。剩下 7 的是幸运数。例如，219->18->7 或者 118->7。

条件：1 <= l <= r <= 1e9

解

先找出小于 1e9 的所有幸运数，再数区间 [l, r] 内幸运数的个数。

幸运数如何计算？逆向构造。例如 219->18，反之通过 18 也可以构造出 219 这个幸运数。

注意 k 位的幸运数不仅可以通过 k-1 位的幸运数计算得到，还可以通过更短的幸运数在前方补 0 后计算得到。例如：7 补成 07，可以计算得到 118, 229, 770, 881, 992。

整数分解

更新于 2020-07-29 分类于 编程

给定 N 和 K，求任意一组互不相同的正整数 x, y, z 满足

  • x + y + z = N
  • gcd(x, y) = gcd(x, y) = gcd(y, z) = K

条件：1 <= N, K <= 1e18

解

原题可转化为：给定 M，求任意一组 a, b, c 使得 a + b + c = M 且 a, b, c 互质。

算法：

  1. 首先计算 M = N/K，若 M 不是整数，则原题无解。
  2. 若 M 是偶数，则可以构造出一组解 a = 1, b = M/2, c = M/2 - 1。（a, b, c 互不相等，M 最小为 6）
  3. 若 M 是奇数，则不断尝试：固定 a = 1, 则 b + c = M - 1 为偶数，若想 gcd(b, c) = 1，b, c 必须都为奇数。所以，令 b = 3, 5 ... (M-1)/2，计算 gcd(b, c)，若结果为 1，则找到一组满足条件的解。
