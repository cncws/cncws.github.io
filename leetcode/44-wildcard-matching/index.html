通配符匹配

给定一个字符串 (s) 和一个字符模式 (p) ，实现一个支持 '?' 和 '*' 的通配符匹配。

两个字符串完全匹配才算匹配成功。

说明:

  • s 可能为空，且只包含从 a-z 的小写字母。
  • p 可能为空，且只包含从 a-z 的小写字母，以及字符 ? 和 *。

递归

递归方法 match(i, j) 判断 s[i:] 是否匹配 p[j:]。

  • 当 p[j] == s[i] or p[j] == '?' 时，s[i] 与 p[j] 只能一对一匹配，应返回 match(i + 1, j + 1) 的结果。
  • 当 p[j] == * 时，* 可能匹配 0 个，也有可能匹配 1 个字符，应返回 match(i, j + 1) or match(i + 1, j) 的结果。
  • 上述两种情况之外，无法匹配，返回 False。

动态规划

上述递归方法可看作自顶向下的动态规划，将其转成自底向上的动态规划即可。

令 dp[i][j] 表示 s[i:m] 是否匹配 p[j:n]。

初始：

空字符串匹配空字符串：dp[m][n] = True。 空字符串匹配只包含 '*' 的模式字符串：dp[m][j] = True if all(p[k] == '*' for k in range(j, n))。

转移方程：

  • 若 p[j] == s[i] or p[j] == '?'，dp[i][j] = dp[i + 1][j + 1]。
  • 若 p[j] == '*'，dp[i][j] = dp[i + 1][j] or dp[i][j + 1。
  • 上述两种情况之外，dp[i][j] = False。

回溯算法

匹配的难点在于不知道 '*' 该匹配多少个字符。此方法中使用一个变量 ii 记录 '*' 匹配的字符串终点。当 s[i] 不匹配 p[j] 时，增加 '*' 匹配的字符个数，即 ii += 1，之后将 i 和 j 移动到合适的位置。
