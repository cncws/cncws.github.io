原理：

设失配时 S 的索引为 i，W 的索引为 j，但 S[i-j:i] 与 W[:j] 是匹配的。为什么下一步能比较 S[i] 与 W[next[j]]。

这与 next[j] 的含义有关：next[j] 是 W[:j] 的最长公共前后缀的长度（小于 W 的长度）。

当我们比较 S[i] 与 W[next[j]] 时，相当于向右滑动 W 使得 W 的前 next[j] 个字符与滑动前的后 next[j] 个字符重叠。

动态规划求 next 数组

使用动态规划可以在线性时间内计算出 next 数组。

初始时 next[0] = -1

假设 next[i - 1] = j，即 W[:i] 的最长公共前后缀的长度为 j，求 next[i]。

  1. 若 W[i] == W[j]，则 next[i] = next[i - 1] + 1

  2. 若 j == -1 or W[i] != W[j]：

    不能直接令 next[i] = 0，而是应该逐步减小前后缀的长度，直到前后缀再次相等。幸运地是，我们不用“逐步”减小，可以利用 next 数组的结果：将后缀看作 needle，将前缀看作 haystack。当“后缀” W[i] 不匹配“前缀” W[j] 时，下一步直接比较 W[i] 与 W[next[j]]（在代码中表现为令 j=next[j]，在下一个循环中再次比较 W[i] 与 W[j]），因为 next 的含义保证了 W[:next[j]] == W[i-next[j]:i]。
