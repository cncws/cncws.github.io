正则表达式匹配

给你一个字符串 s 和一个字符规律 p，请你来实现一个支持 '.' 和 '*' 的正则表达式匹配。

所谓匹配，是要涵盖 整个 字符串 s 的，而不是部分字符串。

说明:

  • s 可能为空，且只包含从 a-z 的小写字母。
  • p 可能为空，且只包含从 a-z 的小写字母，以及字符 . 和 *。

递归

要验证 s[0:] 与 p[0:] 是否匹配，分情况讨论：

  1. p[0:] 为空，则只有当 s[0:] 也为空时，二者匹配。
  2. p[1] == '*'，即 p[0:2] 是形如 x* 的字符串，继续分两种情况：
    • x* 匹配 0 次，递归验证 s[0:] 与 p[2:] 是否匹配。
    • x* 匹配 1 次，此时要求 s[0] 与 p[0] 匹配，即 p[0] == s[0] or p[0] == '.'，然后递归验证 s[1:] 与 p[0:] 是否匹配。
  3. p[1] != '*'，则 s[0] 与 p[0] 只能一对一匹配：p[0] == s[0] or p[0] == '.'，然后递归验证 s[1:] 与 p[1:] 是否匹配。

动态规划

动态规划的逻辑与递归完全相同。通过增加一个 dp 哈希表，缓存已经计算过的结果，在出现相同的计算时直接从表中返回结果。

最便利的方法是使用 @functools.lru_cache 装饰器，装饰 recursive 方法。

自底向上的动态规划

上述动态规划可看作自顶向下求解，需要的结果是 recursive(0, 0)，通过递归的方式计算 recursive(0, 0) 所需的依赖，例如 recursive(0, 2) 等等，在计算 recursive(0, 2) 又会计算它所需的依赖……

我们可以直接自底向上求解：定义 dp[i][j] 为 s[i:] 与 p[j:] 的匹配结果。

初始有：

  • dp[ls][lp] = True，因为此时 s[ls:] == p[lp:] == ''。
  • dp[i][lp] = False, i < ls，因为此时 s[i:] != '' 但 p[lp:] == ''。

dp[i][j] 的更新方式与递归的逻辑完全相同，我们只需将调用 recursive 方法改为更新 dp 数组。
