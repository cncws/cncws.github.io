字典序中第 k 小的单词

更新于 2020-08-09 分类于 编程

n 个 'a' 和 m 个 'z' 组成的单词按照字典序排列。问第 k 个单词是什么。

输入描述：

第一行三个整数 n, m, k( 1 <= n, m <= 100, 1 <= k <= 109), 以空格分割。

输出描述：

输出第 k 个字典中的字符串，如果无解，输出 -1。

这种计算字典序中第 k 小的问题，显然不应该直接求解，即使 k 看上去挺小。

这里同样构造 Trie Tree，例如例子中 n = m = 2 时的树为：

树还是有点区别的，因为字母出现的次数是有限制的。但我们同样可以计算出，一颗子树下面有多少有效字符串。以第一个节点 a 为例，因为第一个字母固定，剩下的字母只能由剩下的 1 个 'a' 与 2 个 'z' 任意组成。因此，有效字符串树是它们的组合数，根据组合数的计算方法 C(3, 1) = 3，因此第一个节点有 3 个字符串。

而我们要找的第 6 个字符串显然不在这个子树的范围内。由此我们可以断定，第一个字母不为 'a'，应该为 'z'。接着，我们在第二个节点为 'z' 的子树中寻找第 6 - 3 = 3 个字符串……

如果我们要找第 2 个字符串，则我们可以断定第一个字母为 'a'，接下来需要下到 'a' 的子树中，寻找第 2 个字符串……

通过这两道题，可以总结出求字典序中第 k 小的单词的步骤：

  1. 虚构出 Trie Tree。
  2. 从第一个节点开始（第一个节点的值就是第一个字符的值），计算出子树中包含的字符串数 X。
  3. 若 X >= k，则说明想求的字符串位于子树中，下到子树中，从子树中的第一个节点开始，执行与第二步类似的操作。
  4. 若 X < k，则说明想求的字符串位于子树外，移动到相邻子树，此时应该求第 k - X 个字符串。

有一些边长为 1 立方体搭了一些塔。这些塔的不稳定值为它们之中最高的塔与最低的塔的高度差。

现进行如下操作：每次从某座塔上取下一块立方体，并把它放到另一座塔上。

求进行了不超过 k 次操作之后，不稳定值最小是多少。

输入描述：

第一行两个数 n,k (1 <= n <= 100, 0 <= k <= 1000) 表示塔的数量以及最多操作的次数。

第二行 n 个数，a[i](1 <= ai <= 104) 表示第 i 座塔的初始高度。

输出描述：

第一行两个数 s, m，表示最小的不稳定值和操作次数 (m <= k)

接下来 m 行，每行两个数 x,y 表示从第 x 座塔上取下一块立方体放到第 y 座塔上。

解

每次操作从最高的塔取下一块立方体，放到最低的塔上。还要注意到，若它们的高度差为 1，则此操作没有意义。应提前退出循环。

我们可以先将塔按高度排序（还需要保存塔的编号信息），之后每次从最末端的塔取下一块立方体，放到最前端的塔上，然后更新数组使之保持有序即可。

但既然 n 的值比较小，每次循环遍历两遍分别找出最大值和最小值也无伤大雅。这里牺牲时间使代码更简洁。
