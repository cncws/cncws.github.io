路由器

一条直线上等距离放置了 n 台路由器。路由器自左向右从 1 到 n 编号。第 i 台路由器到第 j 台路由器的距离为 |i-j|。每台路由器都有自己的信号强度，第 i 台路由器的信号强度为 ai。所有与第 i 台路由器距离不超过 ai 的路由器可以收到第 i 台路由器的信号（注意，每台路由器都能收到自己的信号）。问一共有多少台路由器可以收到至少 k 台不同路由器的信号。

输入描述:

输入第一行两个数 n, k（1 <= n, k <= 10^5）

第二行 n 个数，a1, a2, a3, ..., an（0 <= ai <= 10^9）

输出描述:

输出一个数，一共有多少台路由器可以收到至少 k 台不同路由器的信号。

示例 1

※ 差分数组

首先简单说一下蛮力法：每台路由器维护一个变量 s[i]，表示能接收到 s[i] 种不同的信号。遍历所有路由器，对辐射范围内的所有路由器（包括自己）的变量加 1。最后统计有多少台路由器的 s[i] >= k 即可。

使用差分数组是对蛮力法的一种改进。差分数组为原数组中每一项与前一项的差，例如，原数组为 [a1, a2, a3]，则对应的差分数组为 [a1, a2-a1, a3-a2]。知道了差分数组，在遍历差分数组时就能还原原数组。

差分数组与这道题的联系在哪呢？在蛮力法中，我们关注辐射范围内的每台路由器，事实上，我们只需要关心辐射的边界即可。比如，某台路由器的辐射边界为 l, r，那么当 i 经过 l 后，就能多收到一种信号；经过 r 后，就少收到一种信号。

因此，使用一个数组 d[i] 表示经过边界 i 后信号数量的变化。初始时信号数量为 cur = 0，依次遍历路由器，没经过一个位置 i，信号量变为 cur = cur + d[i]，即知道了当前路由器能收到几种不同的信号。

d[i] 的计算方式也很简单：根据路由器的位置 i 和辐射范围 r，计算出路由器的辐射边界 i - r, i + r。对于左侧边界，d[max(0, i - r)] += 1；对于右侧边界 d[min(i + r + 1, n - 1)] -= 1。

外卖满减

你打开了美了么外卖，选择了一家店，你手里有一张满 X 元减 10 元的券，店里总共有 n 种菜，第 i 种菜一份需要 A[i] 元，因为你不想吃太多份同一种菜，所以每种菜你最多只能点一份，现在问你最少需要选择多少元的商品才能使用这张券。

输入描述:

第一行两个正整数 n 和 X，分别表示菜品数量和券的最低使用价格。（1 <= n <= 100, 1 <= X <= 10000）

接下来一行 n 个整数，第 i 个整数表示第 i 种菜品的价格。（1 <= A[i] <= 100）

输出描述:

一个数，表示最少需要选择多少元的菜才能使用这张满 X 元减 10 元的券，保证有解。

示例 1

动态规划

令 dp[i] 为“使订单价格超过 i 元，最少需要点多少钱的菜“。

对每种菜 j，我们有两种选择：

  • 点，则离目标 i 还差 i - A[j] 元：dp[i] = A[j] + dp[i - A[j]]
  • 不点：dp[i] = dp[i]

二者取较小值：dp[i] = min(dp[i], A[j] + dp[i - A[j]])，这就是此问题的转移方程。

还需要注意的点有：

  1. 数组越界问题：i - A[j] 小于零时，点这一个菜就够 i 元了，所以 dp[i] = min(dp[i], price)。
  2. 每种菜最多只能点一次。这一点体现在必须逆序计算 dp[i]。（0-1 背包问题）

图的遍历

给定一张包含 N 个点、N-1 条边的无向连通图，节点从 1 到 N 编号，每条边的长度均为 1。假设你从 1 号节点出发并打算遍历所有节点，那么总路程至少是多少？

输入描述:

第一行包含一个整数 N，1 <= N <= 10^5。

接下来 N-1 行，每行包含两个整数 X 和 Y，表示 X 号节点和 Y 号节点之间有一条边，1 <= X，Y <= N。

输出描述:

输出总路程的最小值。

示例 1

深度优先搜索

N 个节点 N-1 条边组成的连通图一定是无环的。这对求遍历所有节点的最短路程很重要：不用考虑访问次序的影响。

首先思考“从 1 号节点，遍历所有节点并回到 1 号节点的最短路程” 这一问题。由于图是无环的，其实无论如何遍历，总路程都是 2 * (N - 1)：每条边走两遍。

那不需要回到 1 号节点，最短路程会是多少？答案是，尽量不要走回头路：在最长的路径上（1 号节点到遍历的最后一个节点）不走回头路。

所以问题的解法是：用深度优先搜索找出一条最长的路径，设其长度为 p，2 * (N - 1) - p 就是问题的解。

孙悟空的徒弟

打败魔人布欧以后，孙悟空收了 n 个徒弟，每个徒弟战斗力各不相同。他教导所有的徒弟和体术，合体后战斗力为原战斗力相乘。任何两个徒弟都可以合体，所以一共有 n*(n-1)/2 种合体徒弟。有一天，他想考验一下孙悟天战斗力如何，希望在所有 n*(n-1)/2 种合体徒弟中选择战斗力第 k 高的，与孙悟天对战。可是孙悟空徒弟太多了，他已然懵逼，于是找到了你，请你帮他找到对的人。

输入描述:

第一行两个 int。徒弟数量：n <= 1*10^6；战斗力排名：k <= n*(n-1)/2

第二行空格分隔 n 个 int，表示每个徒弟的战斗力。

输出描述:

战斗力排名k的合体徒弟战斗力。

示例 1

二分查找

题意：有 n 个数，两两组和有 n*(n-1)/2 种，组合后的大小为两数的乘积（例如，2 和 3 组合得到 6）。问第 k 大的组合乘积是多少？

我们不知道第 k 大的组合乘积是多少，也不好直接求。因此，换一种思路：给出一个数 mid，问乘积比 mid 大的组合个数有多少？

  • 若大于等于 k 个，那么增加 mid 的值，再次尝试。
  • 若小于 k 个，那么减小 mid 的值，再次尝试。

类似的问题有：分田地

下面来看如何找出乘积比 mid 大的组合个数。（与计算的个数有些相似）

先将原数组排序，准备两个指针 i, j 分别指向头尾。计算 nums[i] * nums[j] 的值：

  • 若 nums[i] * nums[j] < mid，i = i + 1。
  • 若 nums[i] * nums[j] >= mid，此时组合 (i, j) 的乘积大于 mid，而且 (i+1, j) ... (j-1, j) 等组合的乘积也大于 mid。这样我们就知道了有 j - i 个组合的乘积大于 mid。

下一步，j = j - 1，因为 nums[j] 变小了，所以 i 不用从头开始，接着上一次的位置，继续判断 nums[i] * nums[j] 与 mid 的大小关系……直到 i, j 相遇时结束。
