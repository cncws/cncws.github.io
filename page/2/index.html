二分查找

时间复杂度必须是 O(log n)，那必须是二分查找。一个有序数组经过一次旋转会变成两段各自升序的子数组，且第一段中的最小数字大于第二段中的最大数字。

动态规划

定义 dp[i] 为“以 s[i] 结尾的最长有效括号的长度“。

  • s[i] == '(' 时，dp[i] = 0。
  • s[i] == ')' 时，有效括号的形式有两种：
    • x() 型：前一个字符是 (，此时最后一个右括号与前一个左括号组成一对，dp[i] = 2 + dp[i - 2]。
    • x(y) 型：前一个字符是 )，此时最后一个右括号与 i - dp[i - 1] - 1 组成一对，dp[i] = 2 + dp[i - 1] + dp[i - dp[i - 1] - 2]。

31. 下一个排列

更新于 2020-09-11 分类于 编程 ， 力扣 阅读次数：

下一个排列

实现获取下一个排列的函数，算法需要将给定数字序列重新排列成字典序中下一个更大的排列。

如果不存在下一个更大的排列，则将数字重新排列成最小的排列（即升序排列）。

必须原地修改，只允许使用额外常数空间。

以下是一些例子，输入位于左侧列，其相应输出位于右侧列。

1,2,3 → 1,3,2

3,2,1 → 1,2,3

1,1,5 → 1,5,1

一遍扫描

要使一个排列比当前排列大，只需将后面的较大数字与前面的较小数字交换。

要求下一个较大排列，必须令交换后增加的幅度尽可能小。“交换”的步骤为：

  1. 从后向前扫描，找到第一个较小数字，即 nums[i] < nums[j], i < j。（此时 i 之后的元素是降序的）
  2. 从后向前扫描，找到第一个比 nums[i] 大的数字 nums[j]，将 nums[j] 与 nums[i] 交换，“增加的幅度”最小。
  3. nums[i] 变大后，不论 i 之后的数字如何排列，都会大于原排列。因此要使增加的幅度最小，还需要将 i 之后的数字按升序排列。（此时 i 之后的元素依然是降序的，要将其转为升序，只需要将这一段逆序即可）
