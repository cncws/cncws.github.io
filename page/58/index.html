整数与 IP 地址间的转换

原理：ip 地址的每段可以看成是一个 0-255 的整数，把每段拆分成一个二进制形式组合起来，然后把这个二进制数转变成一个长整数。

举例：一个 ip 地址为 10.0.3.193

组合起来即为：00001010 00000000 00000011 11000001，转换为 10 进制数就是：167773121，即该 IP 地址转换后的数字。

输入描述:

1 输入IP地址

2 输入10进制型的IP地址

输出描述:

1 输出转换成10进制的IP地址

2 输出转换后的IP地址

孙悟空的徒弟

打败魔人布欧以后，孙悟空收了 n 个徒弟，每个徒弟战斗力各不相同。他教导所有的徒弟和体术，合体后战斗力为原战斗力相乘。任何两个徒弟都可以合体，所以一共有 n*(n-1)/2 种合体徒弟。有一天，他想考验一下孙悟天战斗力如何，希望在所有 n*(n-1)/2 种合体徒弟中选择战斗力第 k 高的，与孙悟天对战。可是孙悟空徒弟太多了，他已然懵逼，于是找到了你，请你帮他找到对的人。

输入描述:

第一行两个 int。徒弟数量：n <= 1*10^6；战斗力排名：k <= n*(n-1)/2

第二行空格分隔 n 个 int，表示每个徒弟的战斗力。

输出描述:

战斗力排名k的合体徒弟战斗力。

示例 1

二分查找

题意：有 n 个数，两两组和有 n*(n-1)/2 种，组合后的大小为两数的乘积（例如，2 和 3 组合得到 6）。问第 k 大的组合乘积是多少？

我们不知道第 k 大的组合乘积是多少，也不好直接求。因此，换一种思路：给出一个数 mid，问乘积比 mid 大的组合个数有多少？

  • 若大于等于 k 个，那么增加 mid 的值，再次尝试。
  • 若小于 k 个，那么减小 mid 的值，再次尝试。

类似的问题有：分田地

下面来看如何找出乘积比 mid 大的组合个数。（与计算的个数有些相似）

先将原数组排序，准备两个指针 i, j 分别指向头尾。计算 nums[i] * nums[j] 的值：

  • 若 nums[i] * nums[j] < mid，i = i + 1。
  • 若 nums[i] * nums[j] >= mid，此时组合 (i, j) 的乘积大于 mid，而且 (i+1, j) ... (j-1, j) 等组合的乘积也大于 mid。这样我们就知道了有 j - i 个组合的乘积大于 mid。

下一步，j = j - 1，因为 nums[j] 变小了，所以 i 不用从头开始，接着上一次的位置，继续判断 nums[i] * nums[j] 与 mid 的大小关系……直到 i, j 相遇时结束。
